# 운영 매뉴얼

## 일상 업무 흐름

### 본사 (hq_admin)

1. **오전**: 대시보드에서 신규 발주 확인 → 주문관리에서 검토 후 **확정** (일괄 확정 가능)
   - 확정 즉시 ERP 전송 큐에 등록되고 크론이 이카운트로 전송
2. **피킹관리**: 출고일 선택 → 품목별 합산 리스트로 창고 작업 지시 (인쇄 버튼 → A4)
   - 수량 부족 시 피킹수량에 실제 수량 입력 → 자동으로 부분출고 처리
3. **출고·배송**: 피킹 완료 주문에 기사/차량/운송장 입력 → 출고 확정 → 배송 완료 처리
4. **이카운트 연동 메뉴에서 ERP 실패 건 확인** — 실패 원인(거래처코드 없음 등) 해결 후 재전송

### 창고 (warehouse)

- 로그인하면 피킹관리로 바로 이동. 피킹 시작/완료, 피킹수량 입력, 출고 처리만 가능.

### 가맹점 온보딩

1. 가맹점관리 → 새 가맹점 등록 (고객코드, 이카운트 거래처코드, 최소주문금액, 배송요일, 마감시간)
2. 가맹점 상세 → 취급상품·단가 탭에서 판매할 상품 체크 + 전용 단가 입력
3. 사용자·권한 → 가맹점 계정 생성 (역할: 가맹점주) → 아이디/비밀번호 전달
4. 가맹점은 접속 후 "홈 화면에 추가"로 앱처럼 사용

## 자주 하는 질문

| 상황 | 처리 |
|---|---|
| 가맹점이 주문을 잘못 넣음 | (접수 상태) 가맹점이 취소 요청 → 관리자 승인. 또는 관리자가 주문 상세에서 수량 수정(사유 필수, 이력·알림 자동) |
| 최소금액 미달 예외 허용 | 관리자가 주문 상세에서 확인 후 확정 (placeOrder의 allowBelowMinimum는 관리자 대리주문용) |
| ERP 전송 실패 반복 | 이카운트 연동 메뉴에서 오류 메시지 확인 → 거래처코드/품목코드 매핑 수정 → 재전송 |
| ERP "수동 확인" 상태 | 타임아웃으로 등록 여부 불명확. 이카운트에서 주문번호(비고)로 검색 후, 있으면 재전송하지 말고 완료 처리, 없으면 재전송 |
| 재고가 안 맞음 | 재고현황 → 재고 동기화 버튼 (또는 크론 대기). "최종 재고 반영" 시각 확인 |
| 임시 휴무 | 시스템 설정 → 휴무일 추가 → 해당일 출고 자동 회피 |
| 상품 일시 품절 | 상품관리에서 품절 체크(전체) 또는 가맹점 상세에서 가맹점별 품절 |

## 데이터 내보내기

- 주문관리 → 엑셀(CSV) 다운로드 (필터 적용된 범위, 한글 Excel 호환 BOM 포함)
- 피킹리스트 → 인쇄(A4)

## 백업

- Supabase 대시보드 → Database → Backups (Pro 플랜 일일 자동 백업)
- 수동: `pg_dump` 또는 Supabase CLI `supabase db dump`
